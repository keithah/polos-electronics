  <action>
- Create `layouts/partials/img/picture.html` that:
  - Accepts a dict with keys like: `src`, `alt`, `class`, `width`, `height`, `loading`, `fetchpriority`, and a boolean `publish_only`.
  - Normalizes `src` so callers can pass either `images/foo.jpg` or `/images/foo.jpg` (trim a leading `/`).
  - Loads the source from Hugo Pipes: `resources.Get` (source must be under `assets/`).
  - Produces a WebP variant using Hugo's image processing (use `.Resize` with original width and `webp q70` to bias smaller file size per Phase 10 decisions).
  - Publishes stable, non-fingerprinted URLs by copying outputs to explicit targets:
    - Fallback copy target: same as normalized `src` (e.g. `images/hero-image.jpg`)
    - WebP copy target: same directory/base with `.webp` extension (e.g. `images/hero-image.webp`)
    - Use `resources.Copy` for both.
  - If `publish_only` is true: publish both outputs but render no markup.
  - Otherwise render:
    - `<picture>`
    - `<source srcset="...webp..." type="image/webp">`
    - `<img src="...fallback..." alt=... width=... height=...>` preserving `loading` and `fetchpriority` when provided.

- Update `layouts/index.html` to use this partial for every raster image currently rendered with `<img src="...jpg/png">`:
  - Header logo
  - Hero image
  - Team images (Andy/Derick/John)
  - Services image
  - Contact image
  - Footer logo
  - Service area preview image (uses `data/service_area.json` previewUrl)

- Ensure the map fallback raster (`/images/service-area-map.png`) is still generated even when the interactive embed path is active:
  - Call the partial once with `publish_only=true` for `$mapFallbackUrl` so `public/images/service-area-map.png` and `public/images/service-area-map.webp` always exist.

Constraints:
- Keep OG/Twitter meta tags pointing at `images/hero-image.jpg` (JPG) for crawler compatibility.
- Keep public URLs stable: `.../images/name.jpg` remains, and `.../images/name.webp` is a sibling.
