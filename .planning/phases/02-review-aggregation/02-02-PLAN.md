Transform the existing daily Google-only workflow into a monthly multi-platform workflow:

**Schedule:** Change from daily (`0 0 * * *`) to monthly (`0 0 1 * *` - first day of month at midnight UTC)

**Workflow structure:**

```yaml
name: Fetch Reviews

on:
  schedule:
    - cron: '0 0 1 * *'  # Monthly: first day of month at midnight UTC
  workflow_dispatch:

permissions:
  contents: write

jobs:
  fetch-reviews:
    timeout-minutes: 10
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4
        with:
          token: ${{ secrets.GITHUB_TOKEN }}

      - name: Fetch Google Reviews
        # Fetch via Places API, filter 4-5 stars, transform to normalized schema
        # Store in /tmp/google-reviews.json

      - name: Fetch Yelp Reviews
        # Fetch via Fusion API, filter 4-5 stars, transform to normalized schema
        # Store in /tmp/yelp-reviews.json
        # Note: Yelp API returns only 3 excerpts (160 chars each)

      - name: Merge reviews into data/reviews.json
        # 1. Read existing data/reviews.json
        # 2. Extract manual reviews (homeadvisor, nextdoor) - PRESERVE THESE
        # 3. Merge: manual reviews + new google + new yelp
        # 4. Update platforms metadata (ratings, counts from APIs)
        # 5. Update lastUpdated timestamp
        # 6. Write back to data/reviews.json

      - name: Commit and push if changed
        # Same pattern as existing workflow
```

**Google Reviews step details:**
- Use existing GOOGLE_API_KEY and GOOGLE_PLACE_ID secrets
- Fetch from Places API: `fields=name,rating,reviews,user_ratings_total`
- Filter: `select(.rating >= 4)`
- Transform to normalized schema with `platform: "google"`

**Yelp Reviews step details:**
- Use new YELP_API_KEY secret (add to user_setup)
- Endpoint: `https://api.yelp.com/v3/businesses/polos-electronics-battle-ground/reviews`
- Header: `Authorization: Bearer ${YELP_API_KEY}`
- Filter: `select(.rating >= 4)`
- Transform to normalized schema with `platform: "yelp"`
- Per RESEARCH.md: Returns max 3 excerpts, 160 chars each

**Merge logic (critical):**
```bash
# Extract manual reviews (platforms that don't have API)
MANUAL=$(jq '[.reviews[] | select(.platform == "homeadvisor" or .platform == "nextdoor")]' data/reviews.json)

# Combine: manual + google + yelp
jq -s '{
  reviews: (.[0] + .[1] + .[2]),
  platforms: .[3],
  lastUpdated: now | todate
}' <(echo "$MANUAL") /tmp/google-reviews.json /tmp/yelp-reviews.json <(jq '.platforms' data/reviews.json)
```

**Error handling:**
- If Google API fails: log error, continue with existing Google reviews
- If Yelp API fails: log error, continue with existing Yelp reviews
- Never overwrite manual reviews (HomeAdvisor, Nextdoor)

**Graceful degradation:**
- Check if YELP_API_KEY exists before attempting Yelp fetch
- If missing, skip Yelp step with warning message
