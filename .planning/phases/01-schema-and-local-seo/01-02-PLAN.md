3. Build GeoCoordinates dict from $b.geo fields

   - Nested objects (address, geo, openingHoursSpecification)
   - priceRange, foundingDate
   - image and logo pointing to images/logo.png
   - areaServed array
   - sameAs array
7. Output using: `<script type="application/ld+json">{{ $schema | jsonify | safeJS }}</script>`

Use Hugo's `-` syntax for whitespace control: `{{- ... -}}`

<task type="auto">
  <name>Task 2: Create services.html schema partial</name>
  <files>layouts/partials/schema/services.html</files>
  <action>
Create layouts/partials/schema/services.html following the pattern in RESEARCH.md "Service Schema Array" example.

The partial must:
1. Read services: `{{ $services := .Site.Data.services }}`
2. Get base URL: `{{ $baseURL := .Site.BaseURL }}`
3. Range over $services.services to build Service schema for each:
   - @context: "https://schema.org"
   - @type: "Service"
   - @id: baseURL + "#service-" + .id
   - name: .name
   - description: .description
   - serviceType: .serviceType
   - provider: dict with @id referencing organization (baseURL + "#organization")
   - areaServed: array of @id references to the three county areas from local-business schema

4. Output each service as separate script tag:
```
{{- range $services.services -}}
<script type="application/ld+json">
{{- $service | jsonify | safeJS -}}
</script>
{{- end -}}
```

Services must link back to organization via provider.@id for schema graph connectivity.
  </action>
  <verify>
File exists at layouts/partials/schema/services.html
Contains "Service"
Contains "provider"
Contains "#organization"
  </verify>
  <done>
Services partial exists, iterates over data/services.json, generates Service schema for each with provider reference to organization @id.
  </done>
</task>

   - @id: baseURL + "#faq"

Note: FAQPage is a separate entity from LocalBusiness - it does not need to reference the organization @id.
