<task type="auto">
  <name>Task 2: Export PNG from SVG</name>
  <files>static/images/service-area-map.png</files>
  <action>
Convert SVG to PNG for production use:

Option 1 - Using Inkscape CLI (if available):
```bash
inkscape static/images/service-area-map.svg --export-filename=static/images/service-area-map.png --export-width=800
```

Option 2 - Using ImageMagick (if available):
```bash
convert -background none -density 150 static/images/service-area-map.svg static/images/service-area-map.png
```

Option 3 - Using librsvg (if available):
```bash
rsvg-convert -w 800 static/images/service-area-map.svg > static/images/service-area-map.png
```

Target dimensions: 800px width (height proportional to maintain aspect ratio)

If no CLI tools available, the SVG can be kept as-is since browsers render SVG directly. In that case:
1. Update data/service-area.json to reference .svg instead of .png
2. Note this in the summary for user awareness
