## Directory Purposes

**`.github/workflows/`:**
- Purpose: GitHub Actions automation scripts
- Contains: YAML workflow definitions for CI/CD
- Key files: `hugo.yml` (build/deploy), `fetch-reviews.yml` (data sync)

**`.planning/codebase/`:**
- Purpose: GSD codebase analysis documents
- Contains: Architecture, structure, conventions, testing, concerns, stack, integrations documentation
- Generated by: GSD `/gsd:map-codebase` command

**`archetypes/`:**
- Purpose: Hugo content template boilerplate
- Contains: Template file used when creating new content with `hugo new`
- Key files: `default.md` (unused for this single-page site)

**`assets/css/`:**
- Purpose: Source stylesheets processed by Hugo
- Contains: All CSS styling for the site
- Key files: `custom.css` (1200+ lines, WordPress-matched design)
- Processing: Minified during build via Hugo asset pipeline

**`content/`:**
- Purpose: Markdown source for Hugo content generation
- Contains: Page frontmatter and minimal content (actual content in template)
- Key files: `_index.md` (homepage metadata)

**`data/`:**
- Purpose: Structured data files for Hugo templating
- Contains: JSON data consumed by `index.html` template
- Key files: `reviews.json` (dynamic, updated nightly), `service-area.json` (static)

**`layouts/`:**
- Purpose: Hugo templates that generate HTML from content
- Contains: HTML templates with Go templating syntax
- Key files: `index.html` (complete page template, 651 lines)
- Location used: Hugo renders content through this template

**`static/`:**
- Purpose: Assets served as-is without processing
- Contains: Images, favicons, configuration files
- Key files: `CNAME` (GitHub Pages domain), `images/` (all site graphics)
- Not processed: Files served directly, no Hugo processing

## Key File Locations

**Entry Points:**
- `layouts/index.html`: Primary template; renders complete HTML when Hugo builds
- `hugo.toml`: Hugo configuration; read first by Hugo to understand site structure
- `.github/workflows/hugo.yml`: Build and deployment trigger on git push

**Configuration:**
- `hugo.toml`: Site baseURL, language, title, params (phone, email, address, analytics ID)
- `.github/workflows/hugo.yml`: Build environment (Hugo 0.135.0), minification flags
- `.github/workflows/fetch-reviews.yml`: Review sync schedule (nightly UTC), API secrets config

**Core Logic:**
- `layouts/index.html`: All page structure, navigation, sections, forms, JavaScript
- `assets/css/custom.css`: All styling, responsive breakpoints, component definitions
- `static/CNAME`: GitHub Pages custom domain (poloselectronics.com)

**Data:**
- `data/reviews.json`: Dynamic reviews from Google Places API (structure: name, rating, reviews array)
- `data/service-area.json`: Static service area definition (title, paragraph, cities, map)
- `content/_index.md`: Page metadata (title, description) passed to template

**Testing:**
- No test files in repository (static site, no unit tests needed)
- GitHub Actions workflows serve as integration tests (verify build succeeds)

## Naming Conventions

**Files:**

- **HTML/Templates:** `index.html` (Hugo default for homepage)
- **CSS:** `custom.css` (single stylesheet with all rules)
- **JSON data:** `{noun}.json` (e.g., `reviews.json`, `service-area.json`)
- **Images:** `{section}-{type}.jpg` or `{subject}-name.jpg`
  - Examples: `hero-image.jpg`, `services-image.jpg`, `andy-polos.jpg`, `contact-image.jpg`
- **Markdown:** `_index.md` (Hugo convention for section index)
- **Workflows:** `{purpose}.yml` (e.g., `hugo.yml`, `fetch-reviews.yml`)

**Directories:**

- **Plural collection names:** `assets/`, `layouts/`, `content/`, `data/`, `static/`
- **Semantic subsections:** `css/` (under assets), `images/` (under static), `workflows/` (under .github)
- **Hugo conventions:** `archetypes/`, `layouts/`, `content/`, `data/`, `static/` (Hugo reserved names)

**CSS Classes:**

- **BEM-inspired:** `.site-header`, `.header-top`, `.nav-container`, `.nav-logo`, `.nav-menu`
- **Section naming:** `.hero-section`, `.about-section`, `.services-section`, `.contact-section`
- **Component naming:** `.team-member`, `.review-card`, `.form-group`, `.btn-primary`
- **State classes:** `.active` (for menu open, carousel active dot), `.disabled` (for buttons)
- **Utility classes:** `.section-container` (max-width wrapper), `.form-row` (flex container)

**HTML IDs (Anchor Links):**

- **Section anchors:** `id="home"`, `id="about"`, `id="services"`, `id="service-area"`, `id="contact"`, `id="reviews"`
- **Form elements:** `id="firstName"`, `id="lastName"`, `id="phone"`, `id="email"`, `id="message"`, `id="captcha-answer"`

**JavaScript Variables:**

- **Selectors:** `navToggle`, `navMenu`, `contactForm`, `reviewsCarousel`, `dotsContainer`
- **Numbers:** `cardsPerPage`, `currentPage`, `totalPages`, `num1`, `num2`
- **Strings:** `question`, `answer`, `userAnswer`, `correctAnswer`
- **Arrays:** `cards`, `dots`
- **Functions:** `generateCaptcha()`, `goToPage()`, `nextPage()`, `prevPage()`, `updateCarousel()`

## Where to Add New Code

**New Feature (e.g., Blog Section):**
- HTML markup: Add new `<section>` in `layouts/index.html` with ID and class following pattern
- Styling: Add CSS rules in `assets/css/custom.css` following BEM naming
- Data: If needed, create `data/{feature}.json` for structured content
- Static assets: Place images in `static/images/{feature}/`

**New Component/Module (e.g., Modal Popup):**
- HTML: Add markup to `layouts/index.html` with class `.modal` or similar
- Styles: Add `.modal`, `.modal-*` rules to `assets/css/custom.css`
- JavaScript: Add handler function to `<script>` block in `layouts/index.html`, initialize after DOM ready

**Utilities/Helpers (e.g., Helper Functions):**
- Location: Add to `<script>` block in `layouts/index.html` (no separate utilities file)
- Pattern: Define as named functions (`function helperName() { ... }`)
- Scope: Global functions (no module system, single-page file)

**New Data Source:**
- Location: Create `data/{name}.json` with structure matching other files
- Access in template: Use `{{ .Site.Data.{name}.field }}` syntax
- Update automation: Add workflow step to fetch/transform data if dynamic

**New Static Image:**
- Location: `static/images/{category}/image-name.jpg` or `static/images/image-name.jpg`
- Reference in HTML: Use `{{ "images/image-name.jpg" | relURL }}` to generate correct URL
- Optimization: Compress externally before committing (no build-time optimization)

## Special Directories

**`static/`:**
- Purpose: Static files served unchanged by GitHub Pages
- Generated: No (all files committed)
- Committed: Yes (images, CNAME are version controlled)
- Special handling: `CNAME` file configures custom domain poloselectronics.com

**`assets/`:**
- Purpose: Assets processed by Hugo during build
- Generated: No (source files)
- Committed: Yes (CSS source)
- Special handling: CSS minified via Hugo resource pipeline with `resources.Minify`

**`.github/workflows/`:**
- Purpose: GitHub Actions automation
- Generated: No (manually written YAML)
- Committed: Yes (part of version control)
- Special handling: Secrets (GOOGLE_API_KEY, GOOGLE_PLACE_ID) configured in GitHub Settings

**`public/` (generated, not in repo):**
- Purpose: Hugo build output directory
- Generated: Yes (created by `hugo` command)
- Committed: No (ignored in `.gitignore`)
- Contains: Generated `index.html`, minified CSS, copied images
- Deployed: Uploaded to GitHub Pages by workflow

**`.planning/codebase/`:**
- Purpose: GSD analysis documents
- Generated: Yes (by GSD commands)
- Committed: Yes (part of repo)
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md, STACK.md, INTEGRATIONS.md

## File Interdependencies

```
hugo.toml
  └─> Configures Hugo build
      ├─> Reads: /content/_index.md
      ├─> Reads: /data/*.json
      ├─> Applies: /layouts/index.html
      ├─> Processes: /assets/css/custom.css
      └─> Copies: /static/* to output

.github/workflows/hugo.yml
  └─> Runs: hugo --gc --minify
      └─> Produces: public/index.html (with all styles, content, JS embedded)
      └─> Deploys to GitHub Pages

.github/workflows/fetch-reviews.yml
  └─> Updates: /data/reviews.json
      └─> Triggers GitHub Pages rebuild (via hugo.yml on push)
      └─> Next build includes updated reviews in rendered HTML

layouts/index.html
  ├─> References: hugo.toml params ({{ .Site.Params.* }})
  ├─> References: data/*.json ({{ .Site.Data.* }})
  ├─> References: assets/css/* ({{ resources.Get "css/custom.css" }})
  ├─> References: static/images/* ({{ "images/*.jpg" | relURL }})
  └─> Embeds: Inline JavaScript for interactivity

static/images/*
  └─> Referenced by: layouts/index.html
      └─> Served by: GitHub Pages from static/ directory
```
