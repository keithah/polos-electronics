### Phase 5: Validation & CI Gates

**Goal**: Users are protected from silent homepage regressions because invalid inputs and broken output are blocked before publish
**Depends on**: Phase 4.1
**Requirements**: VALD-01, VALD-02, VALD-03, VALD-04
**Plans**: 2 plans

Plans:
- [x] 05-01: Pre-build validation gates (data contracts, asset existence)
- [x] 05-02: Build hardening and smoke checks (strict mode, error messages)

**Details:**
CI pipeline now validates:
- reviews.json: staleAfterDays (number), freshnessLabel (string), fallbackReviews (array)
- service_area.json: tiers length >= 1
- Assets: service-area-map.svg, service-area-map.png
- Build: --panicOnWarning flag
- Smoke: Reviews section, service area section, map reference
